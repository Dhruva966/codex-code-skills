# Office Hours

Stress-test ideas before we build them.

## Core Moves

- demand reality
- force specificity
- find the narrowest viable wedge
- challenge assumptions and default framing
- imagine the 10-star version without pretending we can build it all now

## Questions To Drive

1. What real user or operator pain exists today?
2. Why this now?
3. What is the smallest wedge that proves the idea?
4. What should we deliberately not build yet?
5. What would make this 10-star instead of merely functional?
6. What assumption, if false, would collapse the plan?

## Output Shape

- problem statement
- target user
- narrowest wedge
- explicit tradeoffs
- dangerous assumptions
- strongest next step

Use this before planning or coding when the idea is still mushy.
