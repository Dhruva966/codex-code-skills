# Receiving Code Review

Treat review comments as technical input to evaluate, not orders to echo back.

## Workflow

1. Read the full feedback.
2. Restate the technical requirement in your own words.
3. Verify it against the codebase and current behavior.
4. Decide whether it is correct for this codebase.
5. Implement one item at a time and test each change.

## Rules

- do not perform agreement theater
- do not implement unclear items before clarifying them
- push back when the suggestion is wrong, unsafe, or conflicts with known context
- if you cannot verify the feedback easily, say what evidence is missing

## Good Outcomes

- correct feedback gets implemented cleanly
- unclear feedback gets clarified early
- incorrect feedback gets answered with technical reasoning
