# Codebase Onboarding

Build a fast mental map before diving into details.

## Workflow

1. Reconnaissance
   - detect manifests, frameworks, entry points, config, tests, and top-level structure
   - use broad search first, not file-by-file reading

2. Architecture map
   - identify stack, app shape, major subsystems, and request/data flow

3. Convention detection
   - naming
   - tests
   - error handling
   - build/run flow
   - git habits when visible

4. Produce an onboarding summary
   - overview
   - key entry points
   - directory map
   - common commands
   - where to look for common tasks

## Good Output

The result should be scannable in a couple of minutes and make the repo feel less mysterious, not more verbose.
